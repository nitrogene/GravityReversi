# Antigravity - Regles de Session

## Workflow Git
* Atomicite : Une fonctionnalite ou une correction = un commit.
* Fin de Session : Chaque session doit se terminer par un git commit et un git push.
* Message : Utiliser les prefixes feat:, fix:, ou docs:.

## Documentation et Maintenance
* README.md : Mise a jour obligatoire lors de l'ajout d'une fonctionnalite majeure ou changement de build.
* Qualite : Compilation avec les flags -Wall -Wextra.

## Etat d'esprit
* Priorite a la stabilite du moteur (Board) avant l'interface.
* Le code poussé sur le depot doit toujours compiler.
* Priorité absolue au respect des règles du jeu Reversi
